## SmartPulse Runtime Config

Optional environment variables for AI-enhanced risk analysis:

- `PORT`: API port (default: `3001`)
- `HOST`: bind host (default: `0.0.0.0`)
- `CORS_ORIGIN`: comma-separated allowed origins (default includes localhost web + `capacitor://localhost`)
- `JWT_SECRET`: signing secret for auth tokens (fallback exists, but set this explicitly in production)
- `HTTPS_KEY_PATH`: path to TLS private key (optional)
- `HTTPS_CERT_PATH`: path to TLS certificate (optional)
- `GROQ_API_KEY`: Groq Cloud API key (enables LLM-generated insights/recommendations)
- `GROQ_MODEL`: Groq model id (default: `llama-3.1-8b-instant`)
- `GROQ_TIMEOUT_MS`: request timeout in milliseconds (default: `1800`)

Without `GROQ_API_KEY`, SmartPulse uses deterministic local fallback insights.
If your frontend runs on `https://localhost`, either configure backend HTTPS (`HTTPS_KEY_PATH` + `HTTPS_CERT_PATH`) or route API through a same-origin HTTPS proxy.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

## Deployment

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

If you are looking for a cloud-based platform to deploy your NestJS application, check out [Mau](https://mau.nestjs.com), our official platform for deploying NestJS applications on AWS. Mau makes deployment straightforward and fast, requiring just a few simple steps:

```bash
$ npm install -g @nestjs/mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.
